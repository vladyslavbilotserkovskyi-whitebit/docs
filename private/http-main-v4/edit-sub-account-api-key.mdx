### Edit Sub-Account API Key

<ApiEndpoint method="POST" path="/api/v4/sub-account/api-key/edit" />

This endpoint updates an existing sub-account API key

‚ùó Rate limit 1000 requests/10 sec.

**Response is cached for:**
NONE

**Parameters:**

| Name     | Type   | Mandatory | Description                                    |
|----------|--------|-----------|------------------------------------------------|
| apiKeyId | String | **Yes**   | ID of the API key to update                   |
| title    | String | **Yes**   | New title for the API key                     |
| urls     | Array  | **Yes**   | Array of URL objects for API key restrictions |

**Request BODY raw:**

```json
{
  "apiKeyId": "a1b2c3d4-e5f6-g7h8-i9j0-k1l2m3n4o5p6",
  "title": "Updated Trading Bot Key",
  "urls": [
    {
      "url": "/api/v4/main-account/address",
      "enable": true
    },
    {
      "url": "/api/v4/main-account/fiat-deposit-url",
      "enable": true
    },
    {
      "url": "/api/v4/main-account/create-new-address",
      "enable": true
    },
    {
      "url": "/api/v4/main-account/withdraw",
      "enable": false
    },
    {
      "url": "/api/v4/main-account/balance",
      "enable": false
    }
    // ... and many more endpoints available in the documentation
  ]
}
```

**Response:**

Available statuses:

- `Status 200`
- `Status 400 if request validation failed`
